# Установка Windows Server 2022 на компьютер с архитектурой arm — Macbook M2 в учебных целях

## Описание эксперимента:
Эти заметки применимы ко всем компьютерам линейки Macbook Air/Pro на базе процессоров M1 M2 M3 архитектуры arm.

Мой Macbook Air M2 8/256 2022

## Установка через UTM

Необходимые приложения/библиотеки/связи:
- Установить brew
   Команда для выполнения в терминале:
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
   Детальная инструкция по установке есть на [официальном сайте](https://brew.sh/).

- Установить инструменты разработчика xcode
   Команда для выполнения в терминале:
```bash
xcode-select --install
```    
- Установить UTM
   Команда для выполнения в терминале:
```bash
brew install utm
```
- Установить qemu
   Команда для выполнения в терминале:
```bash
brew install qemu
```

 Скачать образ Windows Server 2022 можно [здесь](https://www.microsoft.com/fr-fr/evalcenter/download-windows-server-2022).

## Установка Windows Server

![Меню установки UTM](2024-04-06_11-32-38.webp){: .shadow }

Выбираем процесс эмуляции, а не виртуализации, т.к. мы планируем использовать образ системы, созданный для другой архитектуры.

![Меню установки UTM](2024-04-07_16-43-40.webp){: .shadow }

Не выбирайте «другую систему» — будем устанавливать с преднастройками для Windows 10/11. Несколько раз пробовал устанавливать через «Autre», но встроенный биос даже не хотел запускаться или видеть образ системы. Поэтому прислушайтесь к моему опыту и выбирайте Windows 10/11.

![Меню установки UTM](2024-04-07_16-44-45.webp){: .shadow }

Выбираем образ нашей системы — я использовал официальный образ Windows Server 2022 с мартовскими обновлениями 2024 года (на дату написания статьи — апрель 2024). Остальные галочки оставляем без изменений — делаем как на скриншоте.

![Меню установки UTM](2024-04-07_16-46-03.webp){: .shadow }

**ОЧЕНЬ ВАЖНО!** В интернете я много читал про эмуляцию Windows Server, и отзывы об этом подходе в основном негативные: медленно и нестабильно. Но в моём случае (повторю: Macbook Air M2 8/256) всё работает на приемлемом уровне, хотя температура во время установки пугает. Мне ведь нужна система для обучения, а не для реального развёртывания датацентра.

>Первый раз, когда я устанавливал WS, мне тоже показалось всё очень медленным и слишком глючным. Но каково было моё удивление, когда выяснилось, что при установке со стандартными параметрами UTM пытается ставить виндовс, эмулируя только одно ядро процессора. 1! Странно, что об этом не говорят, учитывая лаги и тормоза эмулируемой системы.
{: .prompt-warning }

![Диспетчер задач установленной системы со стандартными настройками эмулируемого процессора](2024-04-06_01-21-55.webp){: .shadow }
Работать в такой системе даже в учебных целях было невозможно.

**РЕШЕНИЕ:** Указываем ядра вручную. Для тестовых целей я указал 4 ядра — установка и запуск новой системы прошли намного быстрее. Думаю, на вашем Macbook Pro можно выделить больше ядер.

Оперативной памяти выбираем чем больше — тем лучше. Я выбрал 4 ГБ, но будь у меня макбук с 16 ГБ, выбрал бы 8 ГБ. Логика простая.

![Меню установки UTM](2024-04-07_16-48-52.webp){: width="700" height="400" : .shadow }

Размер постоянного хранилища — исходя из ваших нужд (и возможностей).

После этого установщик предложил выбрать общую папку для файлов (это можно сделать позднее, но в моём случае я создал отдельный каталог для работы с WS).

![Меню установки UTM](2024-04-07_16-51-09.webp){: .shadow }

Установщик резюмирует все введённые настройки для финального подтверждения.
![Меню установки UTM](2024-04-07_16-52-09.webp){: .shadow }

Теперь система появилась в окне выбора в UTM.
![Меню установки UTM](2024-04-07_16-52-49.webp){: .shadow }

Запускаем

При первом запуске меня забросило на такое окно — решается вводом команды

![Меню установки UTM](2024-04-07_16-53-53.webp){: .shadow }

```bash
exit
```

И без изменения настроек биоса — делаем сброс — reset
![Меню установки UTM](2024-04-07_16-54-46.webp){: .shadow }

## «Press any key to load ...»

Нажимаем любую клавишу

И через какое-то время видим привычное окно установки Windows.

![Меню установки UTM](2024-04-07_16-57-23.webp){: .shadow }

### Советы по установке:

Во время установки Windows Server не забудьте указать нужную редакцию системы:
<!-- markdownlint-capture -->
<!-- markdownlint-disable -->
> Небольшие пояснения:
- Desktop Experience даёт графический интерфейс, привычный по настольным версиям **Windows**.
- Версия Datacenter позволяет создавать виртуальные учётные записи (пользователи системы)
{: .prompt-info }
<!-- markdownlint-restore -->

Для моих учебных целей я буду устанавливать Windows Server 2022 (Desktop Experience) с лицензией Datacenter.

А теперь внимание на температуры во время установки:
![Установка UTM](2024-04-06_00-53-47.webp){: .shadow }

После установки появится стандартное окно разблокировки Windows, в котором попросят нажать Ctrl + Alt + Delete, но так как таких клавиш нет, вставлю здесь один ответ с GitHub UTM [ссылка на пост](https://github.com/utmapp/UTM/issues/3413#issuecomment-1001997191) (GoXLd: другое решение — вы всегда можете подключить стандартную WIN-клавиатуру)

![Windows Server 2022](2024-04-07_17-53-27.webp){: .shadow }

Также не забываем, что при первом запуске система должна установить UTM Guest Tool.

![Windows Server 2022](2024-04-05_20-30-11.webp){: .shadow }
Как по мне, запускать Windows Server таким способом на M2 8/256 — полнейший мазохизм. Система с более точно настроенной эмуляцией работает намного лучше (меньше вылетов и зависаний), но всё равно совершенно далека от плавности виртуализации. Кстати, при работе Windows Server 2022 перегрева системы нет — средняя температура макбука держится в пределах 70–80 градусов.
>Возможно, на моём «слабом» компьютере стоит попробовать более старые (менее требовательные к железу) версии
{: .prompt-tip }
Не забудьте извлечь ISO WS
